Debugging

Debugging is the process of searching for interactions the user may have that could cause an error. By debugging we can find bugs and make our website more user-friendly.

We want to be fully aware of the errors so we can make the user have to deal with them less. While programming we want it to fail loudly and be tested rigorously. We qill then gain the users trust in the production version.

  • exceptions: an errors return value that can be accessed for the error details
  • Warning: Error that does not cause the program to crash. Javascript code will continue to run but warnings can cause unexpected issues if not dealt with.
  • Strict mode: Helps us avoid poor programming practices that can cause spaghetti code and errors in our code down the line.

Types of errors

  • System
  • Programmer: evalError, referenceError, rangeError, syntaxError, URIError, InternalError, typeError
  • User (better known as programmer errors noticed by the user)

Feature detection

Used to determine browser support and consists of an if statement inside which we test to see if the method exists. Some browsers support methods that others do not. We need to capture all types of browsers in use to gain the users trust.

Test Driven Development process

Used to thoroughly test our code. We write tests such as: test('2 is prime', () => { expect(isPrime(2)).toBe(true); }); test('10 is not prime', () => { expect(isPrime(10)).not.toBe(true); }); More on TDD
  1. Write test
  2. Write code to pass the test
  3. Refactor the code
  4. Test refactored code
  5. Write more tests for new features
