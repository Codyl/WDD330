Forms

Form Elements

  • Textarea
  • Input - text, button, checkbox, color, radio, etc.
  • Select
  • Button

Information can be processed on the front-end through Javascript. Information processing means traversing forms and data to work with the data. The action attribute of the form is where the page sends the data and takes the user to. Square bracket notation is better than than using a name attribute or number to access the data. ie. const form = document.forms['search']; form.submit() can be used to submit the form. You can listen for the submit button being pressed with this event listener: form.addEventListener ('submit', callback); Callback functions get passed an event which can be accessed for its information by putting it in the parameter list. You can access parts of the form by its name. ie. form.input1 or form.names[0]. You can then edit its value like this: form.input1.value="test"; Buttons work much the same as input of type button.

Form Elements

Javascript validation involves checks on the form before it is submitted. It does not replace form validation on the server side because the user can easily manipulate data on the front-end and cause issues. We can write our Javascript validation like so: form.addEventListener('submit',validate,false); function validate(event) { const text = form.elementName1.value; if (condition) { //preform validation } } event.preventDefault() prevents the form from being submitted. Provide the user with feedback quickly rather than waiting til submission time and the user will be much happier.

Object-oriented programming in JS

There are three main concepts in OOP:

  • encapsulation: hiding how things work from the user
  • polymorphism: Allowing various objects to morph their functions to work slightly differently.
  • inheritance: Having child classes that can draw from the attributes and methods of its parent class.

Class declarations are less error prone than using constructor functions.

Classes can have static methods that are accessed directly by the class rather than the instance of the class. ie. Dice.description()

Every class has a prototype property and a constructor. A prototype property is a live property that can be manipulated and change objects data after they have been created. ie. className.prototype.propertyName The main purpose of the prototype is 'to add any new properties and methods after the class has been declared'. In the chrome editor it can be accessed using __proto__ method. Like other programming languages you can use the keyword extends to create a child class (inheritance). You can override functions like toString to have a more specific use for objects (polymorphism).

Modern Javascript

Libraries

Libraries provide a way to write code faster but adds overhead and may slow down your code.

Modules

Modules can be added by exporting and importing data. Modules are an alternative to libraries because they allow you to create small reusable bits of code rather than including whole libraries.

MVC

  • Model: functionality for creating, reading, updating and deleting
  • View: What is rendered to the screen as HTML
  • Controller: Reacts to events and communication between the model and view.

Templating Languages

Mustache, Pug and Handlebars are Templating languages to name a few. EJS also. EJS lets you basically write Javascript logic in a html file without it appearing in a script. It is evaluated before it is shown to the user.
